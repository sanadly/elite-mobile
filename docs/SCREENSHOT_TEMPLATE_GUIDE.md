## ğŸ¯ Step-by-Step: Creating Screenshots in Figma

### Step 1: Set Up Figma File

```
1. Go to figma.com
2. Create new design file
3. Create frames:
   - Name: "iPhone 6.7 - [Screen Name]"
   - Size: 1290 x 2796px
   - Background: White (#FFFFFF)
```

### Step 2: Take App Screenshots

**On Physical Device (Best):**
```bash
# iOS
1. Open app on iPhone
2. Navigate to desired screen
3. Press Power + Volume Up
4. AirDrop to Mac

# Android
1. Open app on Android phone
2. Navigate to desired screen
3. Press Power + Volume Down
4. Upload to computer
```

**Using Simulator:**
```bash
# iOS Simulator
1. Run app: npm run ios
2. Navigate to screen
3. Cmd + S to save screenshot
4. Find in Desktop

# Android Emulator
1. Run app: npm run android
2. Navigate to screen
3. Click camera icon in emulator toolbar
```

### Step 3: Import to Figma

```
1. Drag screenshot into Figma frame
2. Scale to fit (maintain aspect ratio)
3. Center in frame
4. Lock layer (optional)
```

### Step 4: Add Device Frame (Optional)

```
1. Install "Mockup" plugin (free)
2. Select your screenshot
3. Run plugin
4. Choose "iPhone 15 Pro Max" or similar
5. Apply
```

### Step 5: Add Text Overlay

```
1. Press T for text tool
2. Type headline
3. Font: Alexandria Bold, 48-60px
4. Color: #012856 (navy) or #FFFFFF (white)
5. Position at top or bottom
6. Add subtle background if needed:
   - Rectangle tool (R)
   - Semi-transparent fill
   - Place behind text
```

### Step 6: Export

```
1. Select frame
2. Right panel > Export
3. Add export settings:
   - Format: PNG or JPG
   - Scale: 1x
4. Click Export
5. Repeat for all screenshots
```

## âœ¨ Pro Tips

### Visual Hierarchy
1. **Most important screen first** - Usually home/product grid
2. **Show core value** - What makes your app special
3. **Build narrative** - Screenshots should tell a story
4. **End with CTA** - Last screenshot encourages download

### Text Overlays
- **Keep it short** - 3-5 words max
- **High contrast** - Text must be readable
- **Consistent style** - Same position/size across screenshots
- **Match brand** - Use brand colors and fonts

### Content
- **Use real data** - Not "Lorem Ipsum" or fake names
- **Show quality** - High-resolution product images
- **Include prices** - Shows it's a real shopping app
- **Display success** - Happy states, not errors

### Testing
- **View on device** - Check how it looks on actual phone
- **Ask others** - Get feedback before submission
- **A/B test** - Try different orders/text after launch
- **Update regularly** - Refresh screenshots every 6-12 months
